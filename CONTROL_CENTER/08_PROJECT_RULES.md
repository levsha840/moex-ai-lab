# 08_PROJECT_RULES — MOEX AI LAB

## Основные правила

1. `CONTROL_CENTER` — единственный источник актуального состояния проекта.
2. После каждого релиза проверяются все 10 документов.
3. Каждый релиз поставляется как патч-комплект.
4. Перед commit должны пройти тесты.
5. Основное окружение проекта — Python 3.12 + `.venv`.
6. Основная команда проверки — `python -m pytest`.
7. Интеграционные проверки не должны ломать unit-test прогон.
8. Секреты и токены не коммитятся.
9. `.env` должен оставаться вне Git.

## Git

После успешного релиза:

```powershell
git add .
git commit -m "vX.X Release Name"
git push
```
