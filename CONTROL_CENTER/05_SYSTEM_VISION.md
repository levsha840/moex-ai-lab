# 05_SYSTEM_VISION

MOEX AI LAB — системное видение платформы после архитектурного обновления.

## Миссия

MOEX AI LAB строится не как одиночный торговый бот, а как платформа исполнения и сопровождения алгоритмических стратегий для MOEX.

Цель платформы — систематически находить, проверять, запускать, контролировать и отключать торговые стратегии в дисциплинированном инженерном процессе.

## Главный принцип

Любая стратегия считается непригодной, пока она не доказала обратное.

Стратегия допускается к капиталу только после прохождения полного lifecycle:

```text
Idea
  -> Research
  -> Implementation
  -> Backtest
  -> Realistic Cost Model
  -> WalkForward
  -> Paper Trading
  -> Production
  -> Continuous Supervision
  -> Scale / Disable / Archive
```

## Четыре контура платформы

### 1. Production Core

Отвечает за исполнение решений:

```text
StrategyEngine
  -> PortfolioAllocationEngine
  -> RiskEngine
  -> PaperTradingEngine / BrokerExecutionEngine
```

Требования:

- детерминированность;
- минимальные побочные эффекты;
- dependency injection;
- воспроизводимость решений;
- отсутствие зависимости от исследовательского кода.

### 2. Validation Core

Отвечает за допуск стратегий:

```text
Backtest
  -> Cost Model
  -> WalkForward
  -> OOS Metrics
  -> PASS / FAIL
```

Главная задача Validation Core — чаще говорить `FAIL`, чем `PASS`.

### 3. Research Core

Отвечает за генерацию и проверку гипотез:

- hypothesis registry;
- feature engineering;
- statistical validation;
- strategy discovery;
- future AI-assisted research.

Research Core не имеет прямого доступа к реальному исполнению сделок.

### 4. Operations Core

Отвечает за сопровождение работающих стратегий:

- strategy supervisor;
- деградация метрик;
- drawdown control;
- regime drift;
- audit trail;
- decision logs.

## Strategy Lifecycle

Стратегия должна быть управляемой сущностью, а не просто классом в коде.

Минимальные статусы:

- `research`;
- `backtest`;
- `walkforward`;
- `paper_trading`;
- `production_candidate`;
- `production`;
- `disabled`;
- `archived`.

## Demo-стратегии

Текущие простые RSI/SMA стратегии используются только как smoke-test и pipeline-test.

Они не считаются доходными стратегиями и не должны использоваться как основание для реальной торговли без отдельной статистической проверки.

## Главный KPI платформы

На раннем этапе главным KPI является не доходность, а скорость и качество обучения платформы:

- сколько гипотез проверено;
- сколько отклонено;
- сколько прошло walk-forward;
- сколько прошло paper trading;
- сколько стратегий отключено корректно;
- сколько времени занимает путь от идеи до объективного вердикта.

## Правило развития

Каждый новый модуль должен либо:

1. улучшать способность платформы зарабатывать;
2. улучшать способность платформы отличать рабочие стратегии от нерабочих;
3. снижать риск самообмана, переобучения или операционных ошибок.

Если модуль не делает ничего из этого, его необходимость должна быть поставлена под сомнение.
